Ringkasan Proyek

Tujuan: Meningkatkan kualitas visual dan UX frontend laporBup tanpa mengganti framework yang ada. Tetap menggunakan Bootstrap 5 + Vanilla CSS + PHP templating. Perubahan harus bersifat targeted — tidak rewrite total.

Aplikasi laporBup adalah sistem pelaporan digital untuk Pemerintah Kabupaten Mandailing Natal. Sistem ini melayani tiga jenis pengguna: Administrator, Camat, dan OPD. Frontend saat ini menggunakan dua sistem layout yang berbeda:

Layout A: Simple (simpel-header + simpel-footer)
Layout B: Admin Panel (sidebar + navbar + setting-panel)
  • Layout A — Digunakan oleh: Landing page, Auth login (admin/camat/opd), Dashboard camat, Dashboard OPD, Laporan camat, Laporan OPD, Data pelapor, Profil, Wilayah, Wagateway
  • Layout B — Digunakan oleh: Dashboard admin, Laporan admin OPD, Laporan admin Camat, Cetak laporan, dan semua halaman manajemen admin
Masalah utama yang ditemukan: Ada inkonsistensi desain antara halaman — auth login menggunakan warna ungu/biru AI gradient, landing page menggunakan hitam-biru-oranye, admin panel menggunakan template Bootstrap default. Tiga "desain" berbeda pada satu aplikasi.

Audit: Tipografi

Font InconsistencySemua halaman
3 font berbeda digunakan tanpa sistem: Poppins (landing+simple), Segoe UI (auth), dan system default (admin panel). Harus digantikan dengan satu font: Plus Jakarta Sans atau Inter untuk seluruh aplikasi.
HIGH
Heading PresenceDashboard, form
Heading pada halaman admin menggunakan ukuran h2/h5 Bootstrap default tanpa letter-spacing adjustment. Terasa datar dan tidak berkarakter. Perlu tight letter-spacing pada heading besar.
MED
Number RenderingStatistik
Angka pada kartu statistik (total laporan dll) menggunakan font yang sama dengan body text. Harus menggunakan font-variant-numeric: tabular-nums agar angka terlihat rapi dan aligned.
LOW
Title Case HeadersSemua
Contoh: "Manajemen Laporan OPD", "Informasi Laporan", "Data Tidak Ditemukan!" — gunakan sentence case untuk lebih natural dan modern. "Data tidak ditemukan" (tanpa tanda seru untuk error state).
LOW

Audit: Warna & Surface

AI Gradient FingerprintAuth, Landing
Halaman login menggunakan linear-gradient(135deg, #667eea 0%, #764ba2 100%) — ini adalah "AI gradient" paling umum. Landing page menggunakan gradient biru-oranye yang sama generiknya. Harus diganti dengan warna yang lebih purposeful dan bernuansa pemerintahan.
HIGH
Pure Black BackgroundLanding
Landing page menggunakan #000000 / #1a1a1a sebagai background. Ganti dengan dark navy atau dark slate (#0d1117, #0f172a) yang lebih sophisticated.
MED
Multiple Accent ColorsDashboard admin
Dashboard admin menggunakan 6 warna berbeda untuk ikon statistik: bg-primary, bg-warning, bg-info, bg-success, bg-purple (#6f42c1), bg-pink (#e83e8c). Terlalu berisik. Maksimal 3 warna semantic yang konsisten.
HIGH
Generic Card ShadowsSemua
Box shadow seluruh kartu menggunakan rgba(0,0,0,0.08). Shadow harus ditint sesuai background — kalau background putih: shadow hangat. Gunakan rgba(15,23,42,0.06) bukan pure black opacity.
LOW
Flat SurfacesSemua
Semua background flat. Tidak ada tekstur, grain, atau depth. Tambahkan subtle noise overlay pada hero section dan header untuk mengurangi kesan digital yang sterile.
LOW
Inconsistent Light ModeLanding dark + Admin light
Landing page dan auth gelap/dark, tetapi begitu masuk ke dashboard langsung menjadi putih terang total. Transisi ini terasa seperti dua aplikasi berbeda. Harus ada token warna yang konsisten antar surface.
HIGH

Symmetrical Landinglanding/index.php

No BreadcrumbSemua halaman detail/edit

HIGH

Before / After — CSS Kritis

1. Background Landing Page

SEBELUM background: linear-gradient( 135deg, #000000, #1a1a1a ); /* Pure black — terasa murah */
SESUDAH background: #0f172a; background-image: radial-gradient(ellipse at 20% 0%, rgba(29,78,216,.15) 0%, transparent 60%), url("noise.svg"); /* Dark slate bertekstur */

2. Warna Auth Login

SEBELUM background: linear-gradient( 135deg, #667eea 0%, #764ba2 100% ); /* Purple AI gradient — generic */
SESUDAH /* Sama dengan landing — unified */ background: #0f172a; /* Left panel: solid blue tinted */ .left-panel { background: #1e3a8a; background-image: radial-gradient(circle at top right, rgba(29,78,216,.3), transparent); }

3. Stat Card Icon

SEBELUM <div style="width:48px;height:48px; background-color:#6f42c1 !important;"> <i class="fas fa-map-marker-alt"> </i> </div> /* Inline style + !important */
SESUDAH <div class="stat-icon stat-icon--camat"> <i class="mdi mdi-map-marker"></i> </div> /* Di CSS: */ .stat-icon--camat { background: rgba(109,40,217,.1); color: #7c3aed; } /* Semantic, maintainable */

4. CSS Variables — Terpusat

SEBELUM — 3 file berbeda /* simple-header.php */ :root { --primary-blue: #2F58CD; } /* auth/index.php */ :root { --primary-color: #1e3a5f; --secondary-color: #3498db; } /* landing/index.php */ :root { --primary-blue: #2F58CD; --primary-orange: #F59E0B; }
SESUDAH — 1 file /* design-tokens.css — dimuat pertama */ :root { --clr-primary: #1d4ed8; --clr-primary-dark: #1e3a8a; --clr-accent: #d97706; --surface-dark: #0f172a; --text-primary: #0f172a; /* ... semua token di sini */ }

5. Validasi Form

SEBELUM if (uraian.length < 10) { e.preventDefault(); alert('Uraian laporan minimal 10 karakter!'); return false; } /* window.alert — blocking, ugly */
SESUDAH if (uraian.length < 10) { e.preventDefault(); showFieldError( 'uraian_laporan', 'Uraian minimal 10 karakter.' ); return false; } /* Inline error di bawah field */
