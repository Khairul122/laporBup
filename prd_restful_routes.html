1

Latar Belakang & Tujuan

Aplikasi laporBup saat ini telah menggunakan Semantic URLs yang bersih, namun pola pemetaan rute dan pengontrolnya masih berorientasi pada konsep RPC (Remote Procedure Call) di mana URL secara eksplisit mengarah ke aksi tertentu (misalnya, `/laporanOPD/create`, `/laporanOPD/store`, `/laporanOPD/edit?id=5`).

Tujuan dari migrasi ini adalah mentransisikan aplikasi ke standar **RESTful Routing** yang sesungguhnya. Keuntungan utama dari RESTful Routing meliputi:

  • Standardisasi API & URL: Menggunakan struktur URL berbasis resource (misal, `/laporan-opd`) dikombinasikan dengan metode HTTP standar untuk mengekspresikan aksi CRUD secara semantik.
  • Pembersihan Query String: Menghapus kebutuhan akan query string `?id=...` untuk aksi detail/edit/update/delete, menggantinya dengan parameter path dinamis (misal, `/laporan-opd/12/edit`).
  • Skalabilitas & Keamanan: Pemetaan eksplisit berdasarkan verb HTTP (`GET`, `POST`, `PUT`, `PATCH`, `DELETE`) memberikan kontrol keamanan tingkat tinggi pada lapisan routing.
ℹ️
Info: Perubahan ini tidak akan mengubah logika bisnis, query database, maupun tampilan antarmuka (UI). Fokus utama adalah merombak Router, daftar rute, parameter method pada controller, dan link URL pada view.
2

Desain Arsitektur Router Baru

2.1 Pattern Matching dengan Regex

Router saat ini hanya mendukung pencocokan string statis ($routePath === $requestPath). Untuk mendukung parameter dinamis seperti /laporan-opd/{id}, Router akan mengonversi placeholder berformat kurung kurawal menjadi ekspresi reguler (Regex) saat pencocokan rute.

Secara teknis, placeholder seperti {id} akan diganti dengan regex ([^/]+). Pencocokan URL akan dilakukan menggunakan preg_match(). Jika rute cocok, nilai parameter yang ditangkap akan diekstrak dan diteruskan ke method controller sebagai argumen.

2.2 HTTP Method Spoofing

Protokol HTML form di web browser secara native hanya mendukung metode GET dan POST. Agar browser dapat mengirimkan request PUT, PATCH, atau DELETE, kita akan mengimplementasikan **HTTP Method Spoofing**.

Views akan mengirimkan input hidden khusus bernama _method di dalam form POST:

HTML Spoofing Form 
<form action="/laporan-opd/12" method="POST">
    <input type="hidden" name="_method" value="PUT">
    <!-- Form fields -->
</form>

Pada core `Router`, jika terdeteksi request POST dan memiliki field _method, nilai method HTTP internal akan dioverride dengan nilai dari field tersebut.

2.3 Metode Controller Injection

Alih-alih controller mengambil ID dari variable global $_GET['id'] atau $_POST['id'], Router akan menyuntikkan (inject) nilai variabel dinamis tersebut langsung sebagai argumen fungsi:

PHP Controller Method Signature 
public function edit($id) {
    $laporan = $this->laporanOPDModel->getLaporanById($id);
    // ...
}

Spesifikasi Kelas Core Router Baru

Berikut adalah rancangan implementasi file core/Router.php yang dimodifikasi untuk mendukung RESTful route, parameter dinamis, dan HTTP Method Spoofing, tanpa mengandung komentar di dalamnya:

PHP core/Router.php 
<?php

class Router {
    private array $routes = [];

    public function register(string $method, string $path, string $handler): void {
        $this->routes[] = [
            'method' => strtoupper($method),
            'path' => trim($path, '/'),
            'handler' => $handler
        ];
    }

    public function get(string $path, string $handler): void {
        $this->register('GET', $path, $handler);
    }

    public function post(string $path, string $handler): void {
        $this->register('POST', $path, $handler);
    }

    public function put(string $path, string $handler): void {
        $this->register('PUT', $path, $handler);
    }

    public function patch(string $path, string $handler): void {
        $this->register('PATCH', $path, $handler);
    }

    public function delete(string $path, string $handler): void {
        $this->register('DELETE', $path, $handler);
    }

    public function any(string $path, string $handler): void {
        $this->register('ANY', $path, $handler);
    }

    public function dispatch(): void {
        $requestMethod = strtoupper($_SERVER['REQUEST_METHOD']);
        if ($requestMethod === 'POST' && isset($_POST['_method'])) {
            $override = strtoupper($_POST['_method']);
            if (in_array($override, ['PUT', 'PATCH', 'DELETE'])) {
                $requestMethod = $override;
            }
        }

        $requestPath = '';
        if (isset($_GET['controller'])) {
            $controller = $_GET['controller'];
            $action = $_GET['action'] ?? 'index';
            $requestPath = trim($controller, '/') . '/' . trim($action, '/');
        } else {
            $requestUri = $_SERVER['REQUEST_URI'] ?? '';
            $path = parse_url($requestUri, PHP_URL_PATH);
            if (defined('BASE_PATH') && BASE_PATH !== '') {
                if (strpos($path, BASE_PATH) === 0) {
                    $path = substr($path, strlen(BASE_PATH));
                }
            }
            $requestPath = trim($path, '/');
        }

        foreach ($this->routes as $route) {
            $routePath = $route['path'];
            $pattern = preg_replace('/\{([a-zA-Z0-9_]+)\}/', '([^/]+)', $routePath);
            $pattern = '#^' . $pattern . '$#';

            if (preg_match($pattern, $requestPath, $matches)) {
                array_shift($matches);

                if ($route['method'] !== 'ANY' && $route['method'] !== $requestMethod) {
                    continue;
                }

                $this->invoke($route['handler'], $matches);
                return;
            }
        }

        http_response_code(404);
        require 'views/errors/404.php';
    }

    private function invoke(string $handler, array $params): void {
        list($class, $method) = explode('@', $handler);
        $file = 'controllers/' . $class . '.php';
        require_once $file;
        call_user_func_array([new $class(), $method], $params);
    }
}

Dampak pada Controller

Semua controller yang menerima parameter dinamis (seperti ID laporan) harus diubah parameternya agar menerima input argument dari Router, bukan membaca variable query string $_GET['id'] atau POST body.

Contoh Refactoring LaporanOPDController (Aksi Detail & Edit):

DIFF — LaporanOPDController.php 
     public function edit($id = null) {
         $this->requireRole('opd');
-        $id = $_GET['id'] ?? 0;
         $userId = $this->getCurrentUserId();
         $laporan = $this->laporanOPDModel->getLaporanById($id);
DIFF — LaporanOPDController.php 
     public function update($id = null) {
         $this->requireRole('opd');
         if ($_SERVER['REQUEST_METHOD'] !== 'POST' && $_SERVER['REQUEST_METHOD'] !== 'PUT') {
             $this->redirect(route('laporanOPD', 'index'));
         }
-        $id = $_POST['id'] ?? 0;
         $userId = $this->getCurrentUserId();
⚠️
Penting: Untuk method POST/PUT/DELETE, parameter $id opsional tetap dideklarasikan dengan nilai default null ($id = null) agar aman jika Router tidak meneruskan argumen, lalu fallback ke POST/GET tetap dipelihara untuk kelenturan.
6

Dampak pada View & Link Generation

6.1 Penyesuaian Helper route()

Helper global route() di core/helpers.php harus dirombak agar mampu menghasilkan URI RESTful secara dinamis berdasarkan parameter yang diberikan.

Sebagai contoh:

  • route('laporanOPD', 'edit', ['id' => 12]) → menghasilkan /laporan-opd/12/edit
  • route('laporanOPD', 'detail', ['id' => 12]) → menghasilkan /laporan-opd/12
  • route('opd', 'delete', ['id' => 3]) → menghasilkan /opd/3
PHP Desain Baru core/helpers.php - route() 
function route($controller, $action = 'index', $extra = []) {
    $controller = strtolower($controller);
    $action = strtolower($action);
    $id = $extra['id'] ?? $extra['id_laporan'] ?? null;

    $resourceMap = [
        'laporanopd' => 'laporan-opd',
        'laporancamat' => 'laporan-camat',
        'laporanopdadmin' => 'admin/laporan-opd',
        'laporancamatadmin' => 'admin/laporan-camat',
        'laporanopdcetak' => 'laporan-opd-cetak',
        'datapelapor' => 'data-pelapor',
        'wagateway' => 'wa-messages',
        'profile' => 'profiles',
        'opd' => 'opd',
        'kecamatan' => 'kecamatan',
        'desa' => 'desa',
    ];

    $resource = $resourceMap[$controller] ?? $controller;

    $path = $resource;
    if ($action === 'create') {
        $path .= '/create';
    } elseif ($action === 'edit' && $id !== null) {
        $path .= '/' . $id . '/edit';
        unset($extra['id']);
    } elseif ($action === 'detail' && $id !== null) {
        $path .= '/' . $id;
        unset($extra['id']);
    } elseif ($action === 'delete' && $id !== null) {
        $path .= '/' . $id;
        unset($extra['id']);
    } elseif ($action === 'updatestatus' && $id !== null) {
        $path .= '/' . $id . '/status';
        unset($extra['id']);
    } elseif ($action === 'download' && $id !== null) {
        $path .= '/' . $id . '/download';
        unset($extra['id']);
    } elseif ($action !== 'index' && $action !== 'store' && $action !== 'update') {
        $path .= '/' . $action;
    }

    $url = url($path);
    if (!empty($extra)) {
        $url .= '?' . http_build_query($extra);
    }
    return $url;
}

6.2 Penyesuaian Form & AJAX pada Views

Seluruh form edit, delete, dan update di views harus ditambahkan method spoofing. Contoh form edit/update:

PHP / HTML views/laporan-opd/form.php 
<form id="laporanForm" method="POST" 
      action="<?= isset($laporan) ? route('laporanOPD', 'update', ['id' => $laporan['id_laporan_opd']]) : route('laporanOPD', 'store') ?>">
    
    <?php if (isset($laporan)): ?>
        <input type="hidden" name="_method" value="PUT">
    <?php endif; ?>
    
    <!-- Input Fields -->
</form>

Rencana Implementasi & Migrasi

Migrasi ke RESTful Routing akan dibagi menjadi 5 fase terencana untuk meminimalisasi downtime dan mencegah bug regresi:

Fase 1: Core Router Upgrade
Melakukan modifikasi pada core/Router.php untuk mendukung regex pattern matching dan deteksi hidden field _method (method spoofing).
Fase 2: Pembaruan Global Helpers & Router Registration
Menyempurnakan fungsi helper route() pada core/helpers.php untuk mendukung penulisan RESTful, dan melakukan mapping ulang seluruh rute di core/routes.php.
Fase 3: Refactoring Controller Signatures
Melakukan update argument method pada controller yang membutuhkan parameter dinamis dan menghapus pembacaan $_GET['id']/$_POST['id'] yang tidak konsisten.
Fase 4: Refactoring Views (Form & AJAX Link)
Menambahkan hidden field _method pada form edit dan form delete di views, serta memigrasikan AJAX calls di JavaScript untuk menembak endpoint RESTful yang tepat.
Fase 5: Validasi & Cleanup
Menghapus fallback routing lama, menjalankan unit test/automated validation, memastikan kebijakan zero comments terpenuhi, dan merilis versi produksi.

Rencana Pengujian (Verification Plan)

Setelah migrasi selesai, proses pengujian otomatis dan manual berikut wajib dijalankan untuk menjamin kualitas rilis:

1. Pengujian Otomatis (Linting & Engine Check)

  • Linting sintaks PHP secara menyeluruh: Get-ChildItem -Recurse -Filter *.php | ForEach-Object { php -l $_.FullName }. Pastikan tidak ada syntax error.
  • Validasi kepatuhan terhadap Zero Comments Policy di semua file yang dimodifikasi.

2. Pengujian Fungsional (Manual Checklist)

Fitur Utama Metode Pengujian Hasil yang Diharapkan
Login Multi-Role Lakukan login sebagai admin, camat, dan OPD via /login Berhasil diarahkan ke dashboard masing-masing tanpa error session.
Pembuatan Laporan (OPD/Camat) Kirim laporan baru dari form tambah laporan (POST /laporan-opd) Data tersimpan di DB, attachment berhasil terunggah, halaman dialihkan ke daftar laporan.
Penyuntingan Laporan (OPD/Camat) Kirim form edit laporan dengan hidden input _method = PUT Data berhasil diperbarui di database tanpa membuat record baru.
Penghapusan Data (OPD/Kecamatan/Desa) Lakukan trigger tombol hapus (DELETE request) Record terkait terhapus dan redirect berjalan mulus.
Export PDF & Excel Klik tombol unduh laporan di cetak laporan (/laporan/pdf) File PDF/Excel ter-generate dan otomatis terunduh di browser.
