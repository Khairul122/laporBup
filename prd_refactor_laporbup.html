Ringkasan Eksekutif

Dokumen ini merupakan Product Requirements Document (PRD) resmi yang mendefinisikan seluruh kebutuhan teknis dan fungsional untuk refactoring codebase aplikasi laporBup — sistem pelaporan berbasis web yang digunakan oleh Pemerintah Kabupaten Mandailing Natal melalui Dinas Komunikasi dan Informatika.

⚠️
Perhatian: Refactoring ini bukan sekadar kosmetik. Codebase saat ini mengandung pola-pola yang dapat menyebabkan bug laten, kerentanan keamanan, dan kesulitan pemeliharaan jangka panjang.

Latar Belakang

Aplikasi laporBup dibangun menggunakan arsitektur MVC custom berbasis PHP native tanpa framework. Meskipun fondasi arsitekturalnya sudah memadai (BaseController, BaseModel, helpers), terdapat inkonsistensi signifikan dalam implementasi di tiap controller dan model yang perlu distandarisasi.

Ruang Lingkup Refactoring

✅ Dalam Ruang Lingkup
  • Standardisasi semua controller
  • Pembuatan sistem Router terpusat
  • Refactoring model dengan prepared statements
  • Pembuatan BaseController yang lebih kuat
  • Penghapusan seluruh komentar kode
  • Standardisasi penamaan metode dan variabel
  • Konsolidasi logika duplikat
❌ Di Luar Ruang Lingkup
  • Perubahan tampilan / UI frontend
  • Perubahan skema database
  • Migrasi ke framework baru (Laravel, dll)
  • Penambahan fitur baru
  • Perubahan bisnis proses

2.2 Temuan Masalah Kritis

🚨
Masalah-masalah berikut memerlukan penanganan segera karena berpotensi menimbulkan kerentanan keamanan atau bug produksi.
# Masalah Lokasi Tingkat Deskripsi
1 SQL Injection OPDModel.php, WilayahModel.php KRITIS Penggunaan escapeString() dan string interpolasi langsung di query SQL tanpa prepared statements. Model yang menggunakan BaseModel::fetchAll() sudah aman, tetapi sebagian masih pakai $this->db->query($sql) langsung.
2 Duplikasi requireAdmin() Semua controller (12+ controller) KRITIS Metode requireAdmin() identik diduplikasi di OPDController, DataPelaporController, LaporanController, WilayahController, dll. Ini melanggar prinsip DRY dan membuat BaseController yang sudah ada menjadi tidak optimal.
3 Route handling di dalam Controller AuthController.php, DashboardController.php, DataPelaporController.php KRITIS Blok if (isset($_GET['action'])) { switch... } di bagian bawah file controller menyebabkan controller bertanggung jawab atas routing-nya sendiri, melanggar SRP (Single Responsibility Principle).
4 Header duplikat di semua controller Semua controller AJAX TINGGI header('Content-Type: application/json') dan echo json_encode() diulang di setiap metode JSON, bukan memanfaatkan $this->json() dari BaseController.
5 Fungsi global bersarang di dalam method LaporanController.php baris 251 TINGGI Fungsi formatTanggalIndonesia() didefinisikan di dalam method generatePDFFile() (fungsi di dalam fungsi), dan juga ada versi duplikatnya sebagai private method di baris 333. Ini akan menyebabkan fatal error jika metode dipanggil lebih dari sekali dalam satu request.
6 Error reporting dimatikan secara tersembunyi LaporanController.php, AuthController.php TINGGI error_reporting(0) dan ini_set('display_errors', 0) dipanggil di dalam file controller, mengubah setting global secara tersembunyi dan membuat debugging sulit.
7 Hard-coded redirect dengan string URL mentah OPDController.php baris 99, 102, dll TINGGI Penggunaan header('Location: index.php?controller=opd&action=index') langsung tanpa menggunakan helper route() yang sudah tersedia, menyebabkan inkonsistensi.
8 Database query di dalam Controller WilayahController.php baris 328 KRITIS Query SQL langsung dilakukan di dalam controller: $query = "SELECT ... WHERE id_kecamatan = " . (int)$id_kecamatan, melanggar arsitektur MVC dan bypassing Model layer.
9 Akses $this->db di Controller WilayahController.php baris 329 KRITIS $this->db->query() dipanggil dari controller padahal $db adalah properti protected di BaseModel, bukan BaseController. Ini menunjukkan kebocoran abstraksi.

Tujuan Refactoring

Tujuan Utama

  1. Keamanan — Eliminasi semua raw query SQL; seluruh interaksi database wajib menggunakan prepared statements via BaseModel.
  2. DRY (Don't Repeat Yourself) — Logika duplikat seperti requireAdmin(), JSON response, dan redirect dipindahkan ke BaseController.
  3. SRP (Single Responsibility Principle) — Controller hanya menangani HTTP request/response. Routing ditangani oleh Router. Logika bisnis di Model.
  4. Keterbacaan — Kode harus self-documenting tanpa komentar. Nama variabel, metode, dan kelas menjelaskan dirinya sendiri.
  5. Konsistensi — Satu konvensi penamaan, satu pola redirect, satu cara render view.

Prinsip yang Diadopsi

📐 SOLID Principles

Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion — diterapkan sesuai konteks PHP native.

🧼 Clean Code (Robert C. Martin)

Fungsi kecil, nama yang bermakna, tidak ada komentar yang tidak perlu, tidak ada kode mati.

🔒 Defense in Depth

Input validation, prepared statements, role-based access control yang konsisten.

Arsitektur Router Terpusat

4.1 Masalah Router Saat Ini

Sistem routing saat ini terbagi antara index.php (dispatcher) dan blok switch di dalam masing-masing file controller. Ini menyebabkan:

  • Tidak ada satu tempat tunggal untuk melihat semua route yang terdaftar
  • Controller ikut bertanggung jawab atas routing (melanggar SRP)
  • Sulit menambahkan middleware (seperti logging, rate limiting) secara terpusat
  • Nama action tidak divalidasi terpusat — index.php hanya memanggil metode apapun yang ada
// ❌ Saat ini: routing di dalam Controller (AuthController.php baris 255–282) if (isset($_GET['action'])) { $authController = new AuthController(); switch ($_GET['action']) { case 'login': $authController->login(); break; case 'logout': $authController->logout(); break; // ... } }   // ✅ Seharusnya: semua route didefinisikan di satu file (core/Router.php) $router->get('auth', 'index', 'AuthController@index'); $router->get('auth', 'admin', 'AuthController@admin'); $router->post('auth', 'login', 'AuthController@login'); $router->get('auth', 'logout', 'AuthController@logout');

4.2 Desain Router Baru

Router baru dirancang sebagai kelas sederhana yang memetakan kombinasi (controller, action) ke handler ControllerClass@method, dengan dukungan metode HTTP (GET/POST).

PHP — core/Router.php (Kelas Baru) File Baru 
<?php

class Router {

    private $routes = [];

    public function register($method, $controller, $action, $handler): void {
        $this->routes[] = [
            'method'     => strtoupper($method),
            'controller' => $controller,
            'action'     => $action,
            'handler'    => $handler,
        ];
    }

    public function get($controller, $action, $handler): void {
        $this->register('GET', $controller, $action, $handler);
    }

    public function post($controller, $action, $handler): void {
        $this->register('POST', $controller, $action, $handler);
    }

    public function any($controller, $action, $handler): void {
        $this->register('ANY', $controller, $action, $handler);
    }

    public function dispatch(): void {
        $requestController = $_GET['controller'] ?? 'auth';
        $requestAction     = $_GET['action'] ?? 'index';
        $requestMethod     = strtoupper($_SERVER['REQUEST_METHOD']);

        foreach ($this->routes as $route) {
            if ($route['controller'] !== $requestController) continue;
            if ($route['action'] !== $requestAction) continue;
            if ($route['method'] !== 'ANY' && $route['method'] !== $requestMethod) continue;

            $this->invoke($route['handler']);
            return;
        }

        http_response_code(404);
        require 'views/errors/404.php';
    }

    private function invoke($handler): void {
        [$class, $method] = explode('@', $handler);
        $file = 'controllers/' . $class . '.php';

        require_once $file;
        (new $class())->$method();
    }
}

5.2 Standar Controller

📋
Setiap controller wajib mengikuti pola ini. Tidak boleh ada duplikasi logika, raw SQL, atau blok routing di dalam controller.
PHP — Template Controller Bersih 
<?php

require_once 'models/OPDModel.php';

class OPDController extends BaseController {

    private OPDModel $opdModel;

    public function __construct() {
        $this->opdModel = new OPDModel();
    }

    public function index(): void {
        $this->requireRole('admin');

        $page   = (int) ($_GET['page'] ?? 1);
        $search = trim($_GET['search'] ?? '');
        $result = $this->opdModel->getAllOPD($page, 10, $search);

        $this->render('opd/index', [
            'opds'        => $result['data'],
            'total'       => $result['total'],
            'totalPages'  => $result['total_pages'],
            'currentPage' => $page,
            'search'      => $search,
        ]);
    }

    public function store(): void {
        $this->requireRole('admin');

        $namaOpd = trim($_POST['nama_opd'] ?? '');

        if (empty($namaOpd)) {
            $_SESSION['error'] = 'Nama OPD wajib diisi';
            $this->redirect(route('opd', 'create'));
        }

        if ($this->opdModel->namaOpdExists($namaOpd)) {
            $_SESSION['error'] = 'Nama OPD sudah terdaftar';
            $this->redirect(route('opd', 'create'));
        }

        $this->opdModel->createOPD(['nama_opd' => $namaOpd])
            ? $_SESSION['success'] = 'OPD berhasil ditambahkan'
            : $_SESSION['error']   = 'Gagal menambahkan OPD';

        $this->redirect(route('opd', 'index'));
    }

    public function delete(): void {
        $this->requireRole('admin');

        $id = (int) ($_GET['id'] ?? $_POST['id'] ?? 0);

        if (!$id) {
            $this->json(['success' => false, 'message' => 'ID tidak valid']);
        }

        $this->json($this->opdModel->deleteOPD($id));
    }
}

5.3 Standar Model

🔒
Wajib: Semua query database harus menggunakan prepared statements melalui metode fetchAll(), fetchOne(), atau query() dari BaseModel. Penggunaan $this->db->query($sql) dengan string interpolasi langsung dilarang.
PHP — OPDModel Setelah Refactor (Prepared Statements) 
<?php

require_once __DIR__ . '/../models/BaseModel.php';

class OPDModel extends BaseModel {

    public function getAllOPD(int $page, int $limit, string $search): array {
        if ($search !== '') {
            $like   = '%' . $search . '%';
            return $this->paginate(
                'SELECT * FROM opd WHERE nama_opd LIKE ? ORDER BY nama_opd ASC',
                'SELECT COUNT(*) FROM opd WHERE nama_opd LIKE ?',
                's', [$like], $page, $limit
            );
        }

        return $this->paginate(
            'SELECT * FROM opd ORDER BY nama_opd ASC',
            'SELECT COUNT(*) FROM opd',
            '', [], $page, $limit
        );
    }

    public function findById(int $id): ?array {
        return $this->fetchOne('SELECT * FROM opd WHERE id_opd = ?', 'i', [$id]);
    }

    public function createOPD(array $data): bool|int {
        $stmt = $this->query('INSERT INTO opd (nama_opd) VALUES (?)', 's', [$data['nama_opd']]);
        return $stmt->affected_rows > 0 ? $this->db->insert_id : false;
    }

    public function namaOpdExists(string $namaOpd, ?int $excludeId = null): bool {
        if ($excludeId) {
            $row = $this->fetchOne(
                'SELECT COUNT(*) AS c FROM opd WHERE nama_opd = ? AND id_opd != ?',
                'si', [$namaOpd, $excludeId]
            );
        } else {
            $row = $this->fetchOne('SELECT COUNT(*) AS c FROM opd WHERE nama_opd = ?', 's', [$namaOpd]);
        }
        return (int) ($row['c'] ?? 0) > 0;
    }
}

5.4 Aturan Tanpa Komentar

Mengikuti filosofi Clean Code oleh Robert C. Martin: kode yang baik tidak membutuhkan komentar. Komentar adalah tanda bahwa kode belum cukup jelas. Berikut adalah aturan yang berlaku:

❌ Komentar yang Harus Dihapus

  • Komentar yang menjelaskan apa yang jelas dari nama metode:
    // Ambil semua data OPD sebelum getAllOPD()
  • Komentar placeholder / TODO lama
  • Komentar section divider (// ===== AUTH =====)
  • Komentar yang menceritakan history perubahan
  • Komentar yang dimatikan (// $result = ...)

✅ Kode Self-Documenting

  • Nama metode menjelaskan tindakannya: namaOpdExists() vs checkNamaOPDExists()
  • Nama variabel bermakna: $totalPages vs $tp
  • Extract method untuk logika kompleks
  • Guard clauses menggantikan nested if
  • Type hints PHP 8.x untuk dokumentasi tipe
